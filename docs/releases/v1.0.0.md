# gabp-runtime v1.0.0 Release Plan

## GitHub Release Title

`gabp-runtime v1.0.0`

## GitHub Release Notes

`gabp-runtime v1.0.0` is the first stable release of the shared Go and .NET
runtime layer for the Game Agent Bridge Protocol.

This release keeps targeting the `GABP` schema line `1.0`, but it syncs the
latest additive protocol assets from `GABP v1.1.0`. That includes the new
attention schemas and conformance fixtures while staying on the same `gabp/1`
wire version.

### Highlights

- First stable package release for both runtime surfaces:
  - NuGet: `Gabp.Runtime` `1.0.0`
  - Go module tag: `github.com/pardeike/gabp-runtime@v1.0.0`
- Added shared runtime DTOs and protocol constants for:
  - `attention/current`
  - `attention/ack`
  - `attention/opened`, `attention/updated`, and `attention/cleared`
- Synced `testdata/gabp/1.0` from the additive `GABP v1.1.0` release,
  including:
  - attention schemas
  - attention conformance fixtures
  - refreshed copied schema/conformance READMEs
- Cleaned up release-facing documentation so the repo no longer advertises the
  old prerelease alpha package version.

### Verification

- `go test ./...`
- `dotnet test dotnet/Gabp.Runtime.Tests/Gabp.Runtime.Tests.csproj`
- `dotnet pack dotnet/Gabp.Runtime/Gabp.Runtime.csproj -c Release`

## Release Procedure

1. Commit the release-prep changes on `main`.
2. Push `main` to GitHub.
3. Create and push tag `v1.0.0`.
4. Create the GitHub Release for `v1.0.0` using the notes above.
5. Publish the GitHub Release.

Publishing the GitHub Release should trigger the `Publish Gabp.Runtime`
workflow automatically.

## Automation Notes

- NuGet publish is automated by `.github/workflows/publish-nuget.yml`.
- The workflow now verifies:
  - the release tag matches the package version in `Gabp.Runtime.csproj`
  - Go tests pass
  - .NET tests pass
  - the package packs successfully before push
- The repo secret `NUGET_API_KEY` already exists on GitHub.
- Go does not require a separate publish workflow. The `v1.0.0` git tag is the
  release mechanism for the module.

## Post-Release Checks

1. Confirm the `Publish Gabp.Runtime` workflow run completes successfully.
2. Confirm `Gabp.Runtime 1.0.0` appears on NuGet.
3. Confirm `go list -m github.com/pardeike/gabp-runtime@v1.0.0` resolves.
